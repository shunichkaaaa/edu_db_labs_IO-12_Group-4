# Сценарії використання

1. ID User_Registration
НАЗВА Створення облікового запису користувача у системі
УЧАСНИКИ Користувач, Система
ПЕРЕДУМОВИ Обліковий запис користувача попередньо не був створений у системі
РЕЗУЛЬТАТ Створення облікового запису користувача в системі
ВИКЛЮЧНІ СИТУАЦІЇ UserReg_EX_AccAlreadyExists — Обліковий запис користувача вже створений у системі
UserReg_EX_NoInput — Користувач не ввів свої реєстраційні дані
ОСНОВНИЙ СЦЕНАРІЙ 👇
2. ID User_Authorization
НАЗВА Авторизація користувача у системі
УЧАСНИКИ Користувач, Система
ПЕРЕДУМОВИ Обліковий запис користувача попередньо не був створений у системі
РЕЗУЛЬТАТ Користувач авторизований у системі
ВИКЛЮЧНІ СИТУАЦІЇ UserAuth_EX_AccountNotExist — Користувач не зареєстрований у системі
UserAuth_EX_NoInput — Користувач не вписав авторизаційні дані та намагається авторизуватися
UserAuth_EX_InvalidСredentials — Користувач не пройшов авторизацію
ОСНОВНИЙ СЦЕНАРІЙ 👇
3. ID Profile_Edit
НАЗВА Редагування профілю користувача у системі
УЧАСНИКИ Користувач, Система
ПЕРЕДУМОВИ Користувач авторизований у системі
РЕЗУЛЬТАТ Профіль користувача відредагований
ВИКЛЮЧНІ СИТУАЦІЇ ProfileEdit_EX_NotAuth — Користувач не авторизований і намагається перейти до профілю
ОСНОВНИЙ СЦЕНАРІЙ 👇
4. ID Support_Service
НАЗВА Вирішення проблем користувача
УЧАСНИКИ Користувач, Система, Адміністратор системи
ПЕРЕДУМОВИ Користувач має проблему, пов'язану з роботою системи;
Користувач не може знайти рішення за допомогою розділу Help
РЕЗУЛЬТАТ Вирішення проблеми користувача
ВИКЛЮЧНІ СИТУАЦІЇ Support_EX_Cancelled — Натиснута кнопка "Відміна"
ОСНОВНИЙ СЦЕНАРІЙ 👇
5. ID Project_Create
НАЗВА Створення проєкту
УЧАСНИКИ Система, Менеджер воркспейсу
ПЕРЕДУМОВИ Відсутні
РЕЗУЛЬТАТ Проєкт створений у системі
ВИКЛЮЧНІ СИТУАЦІЇ ProjectCreate_EX_NoName — Поле з назвою проєкту не заповнене.
ProjectCreate_EX_SameNameExists — Проєкт з такою назвою вже існує.
ProjectCreate_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
7. ID Member_Add
НАЗВА Приєднання користувача до проєкту
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування проєкту
РЕЗУЛЬТАТ Новий користувач доданий до проєкту
ВИКЛЮЧНІ СИТУАЦІЇ MemberAdd_EX_UserNotExist — Користувача з таким нікнеймом не існує.
MemberAdd_EX_UserIsAlreadyMember — Користувач вже є учасником проєкту.
MemberAdd_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
8. ID Member_Remove
НАЗВА Видалення користувача з проєкту
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування проєкту
Існування учасника проєкту
РЕЗУЛЬТАТ Користувач був усунутий з проєкту
ВИКЛЮЧНІ СИТУАЦІЇ MemberRemove_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
9. ID Board_Create
НАЗВА Створення дошки
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування проєкту
РЕЗУЛЬТАТ Дошка створена в проекті
ВИКЛЮЧНІ СИТУАЦІЇ BoardCreate_EX_NoName – Назву дошки не вказано.
BoardCreate_EX_SameNameExists – Дошка з такою назвою вже існує.
BoardCreate_EX_Cancelled – Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
10. ID Board_Remove
НАЗВА Видалення дошки
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування проєкту
РЕЗУЛЬТАТ Дошка вилучена з проєкту
ВИКЛЮЧНІ СИТУАЦІЇ MemberRemove_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
11. ID Task_Create
НАЗВА Створення завдання для користувача
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування дошки
Існування учасника проєкту
РЕЗУЛЬТАТ Завдання для користувача додане на дошку
ВИКЛЮЧНІ СИТУАЦІЇ TaskCreate_EX_NoName — Назву завдання не вказано.
TaskCreate_EX_NoDescr — Не доданий опис завдання.
TaskCreate_EX_NoUser — Не обраний користувач, якому треба назначити завдання.
TaskCreate_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
12. ID Task_Edit
НАЗВА Редагування завдання
УЧАСНИКИ Система, Менеджер проєкту
ПЕРЕДУМОВИ Існування завдання
РЕЗУЛЬТАТ Параметри завдання відредаговані
ВИКЛЮЧНІ СИТУАЦІЇ Task_EX_Cancelled — Натиснута кнопка "Відміна".
ОСНОВНИЙ СЦЕНАРІЙ 👇
